## How to Apply This

When critiquing a design for Gbemi:
- Lead with usability, not aesthetics
- If something looks good because it works well, say so and why
- If something looks right because the structure is right, name the structure
- Flag accessibility failures as blockers, not suggestions
- Be specific: "The heading hierarchy is missing H2s between sections" beats "the structure could be better"
