# Skill: Ariadne Design Critique

## When to use this skill

When reviewing a UI, screenshot, layout, flow, or visual design.

---

## Core Discipline Pattern

**Orient. Simplify. Improve accessibility.**

This is the irreducible Ariadne critique workflow.

---

## Step 1 — Orient

Where am I? What is this?

- Name the elements visible
- State the hierarchy (what leads, what supports)
- Identify the primary user goal
- Assess whether the layout serves that goal

---

## Step 2 — Simplify

Can the user find what they need without confusion?

- Is the information hierarchy findable?
- Is the flow logical from the user's perspective?
- Are there unnecessary elements competing with the primary goal?
- Is the visual noise level appropriate for the context?

---

## Step 3 — Improve Accessibility

WCAG AA is the minimum bar. Check:

| Check | Minimum |
|---|---|
| Contrast (normal text) | 4.5:1 |
| Contrast (large text) | 3:1 |
| Focus indicators | Visible on keyboard navigation |
| Semantic HTML | Proper heading hierarchy, landmarks |
| Touch targets | Minimum 44×44px |
| Error identification | Text-based, not color alone |

---

## Deliverable Format

For each critique, state:
1. What works and why (brief — don't pad)
2. What doesn't work and why it matters for the user
3. Specific accessibility failures with WCAG reference
4. Recommended action for each finding

---

## Escalation Triggers

Escalate to Nero when:
- An accessibility failure is severe enough to exclude users
- A design choice has business or strategic implications beyond UX
- Something requires Argus to reassess technical risk
