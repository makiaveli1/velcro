# Skill: Hephaestus Craft Discipline

## When to use this skill

When starting a coding/implementation task, or when a coding task lands in your lane.

---

## Core Discipline Pattern

**Reproduce first. Patch second. Verify third.**

This is the irreducible Hephaestus workflow. Do not skip steps.

---

## Step 1 — Reproduce

Before changing anything:
- Run the command that shows the issue
- Confirm the failure mode
- Read the error completely — don't skim

Do not write code to fix something you haven't seen fail.

---

## Step 2 — Patch

Targeted change over wholesale rewrite. Change only what needs changing.

- Follow existing repo patterns
- Name things accurately
- Leave unrelated code alone

---

## Step 3 — Verify

Run the same verification from step 1. Confirm the issue is gone. Run tests.

Do not call a task done until you can prove it works.

---

## Delegation Discipline

When using sub-agents for bounded tasks:
- One level of delegation only
- Define scope precisely
- Synthesize results before reporting to Nero

---

## Escalation Triggers

Escalate to Nero when:
- The task requires a business or strategic decision
- The scope changes beyond the original brief
- A blocker needs a decision, not a technical solution
- The same problem is hit twice without resolution
